# Related Work

## Primary Prior Art

### CodeHalu — arXiv:2405.00253
**"CodeHalu: Investigating Code Hallucinations in LLMs via Execution-based Verification"**
Xu et al., 2024. https://arxiv.org/abs/2405.00253

The closest prior taxonomy to this work. Defines 4 coarse categories:
- **Mapping Hallucination** — wrong API/library references
- **Logic Hallucination** — correct structure, wrong semantics
- **Syntax Hallucination** — invalid Python
- **Reference Hallucination** — undefined variables, wrong types

**Relationship to this repo:** CodeHalu's 4 categories roughly map to our H and PB
categories. This repo adds: agent-specific failures (AT), production-only patterns
(MF, DC, OG, WG, CD), testing environment artifacts (TEC), and the cascade propagation
model (CP). CodeHalu focuses on benchmark evaluation; this repo focuses on production
operational debugging.

---

### HalluCode — arXiv:2401.10509
**"HalluCode: Hallucinations in Code Generation from Large Language Models"**
Liu et al., 2024. https://arxiv.org/abs/2401.10509

Focuses on: fabricated imports, non-existent method calls, wrong signatures.
Subset of our H category (H3 covers fabricated imports, H1 covers non-existent attrs).

---

### EvalPlus — arXiv:2305.01210
**"Is Your Code Generated by ChatGPT Really Correct? Rigorous Evaluation of Large Language Models for Code Generation"**
Liu et al., 2023. https://arxiv.org/abs/2305.01210

Measures functional correctness, not failure taxonomy. The EvalPlus benchmark
(HumanEval+, MBPP+) adds more test cases to reveal edge-case failures.
Their "incorrect" category is what this repo seeks to classify.

**Relationship:** EvalPlus reveals THAT code is wrong; this repo classifies HOW it's
wrong to enable detection and prevention.

---

### SWE-bench — arXiv:2310.06770
**"SWE-bench: Can Language Models Resolve Real-World GitHub Issues?"**
Jimenez et al., 2023. https://arxiv.org/abs/2310.06770

Evaluates LLMs on resolving real GitHub issues in open-source repos.
Most relevant to our CP (cascade propagation) and CD (configuration drift) categories.

## Related Tools

### deepeval
https://github.com/confident-ai/deepeval

LLM evaluation framework. Includes hallucination metrics for RAG pipelines,
not code generation. Their `HallucinationMetric` scores factual consistency.
**Gap:** RAG hallucination != code hallucination — different root cause structure.

### Semgrep
https://semgrep.dev

Rule-based static analysis that can catch H3, H6, SB1, SB2 patterns with
custom rules. Semgrep registry has some LLM-generated code rules.
See: https://semgrep.dev/p/security

**Our rules/** directory provides ruff/AST-based checks as a lighter alternative.

### Bandit
https://bandit.readthedocs.io

Python security scanner. Catches some H6 patterns (deprecated APIs with
security implications) and SB2 (naive datetime in auth contexts).

## Datasets

### BigCode The Stack / StarCoder
https://huggingface.co/datasets/bigcode/the-stack

Deduped source code dataset. Can be mined for instances of H2 (registry collision),
H7 (vacuous tests), SL2 (return-in-fixture) patterns at scale.

### GitHub Copilot Bug Studies
- "An Empirical Study of the Effectiveness of GitHub Copilot" (Imai 2022)
- "Measuring GitHub Copilot's Impact on Productivity" (Kalliamvakou 2022)

Both measure productivity impact, not failure taxonomy.
